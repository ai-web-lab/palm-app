# 04. UI / 画面フロー

プロトタイプ（`prototype/prototype.html`）で実証済みの画面仕様。

## 画面遷移

```
[0] 利き手選択 → [1] 占う手選択 → [2] 撮影 → [3] 結果
                                    ↑   │（両手は片手ずつ2回）
                                    └───┘
```

上部にステップドット（現在地表示）。各画面はカード形式。

---

## [0] 利き手選択

- 「右利き / 左利き」を選ぶ。
- 用途：先天運（非利き手）/ 後天運（利き手）の割り当て。
- コピー例：「利き手を『いまのあなた』、反対の手を『生まれ持った本質』として読み解きます」

## [1] 占う手選択

- 「右手 / 左手 / 両手」を選ぶ。
  - 右手：いまの自分・これから（手軽）
  - 左手：生まれ持った本質・素質
  - 両手：本式・右手と左手を1枚ずつ撮影
- 両手を選んでも、撮影は片手ずつ（→ [2]を2回）。

## [2] 撮影

- カメラ枠（四隅コーナーマーク）の中に手のひらを収める。
- **選んだ手に応じたガイドを表示**：
  - 右手 → 親指が左のガイド。左手 → 鏡像（親指が右）。
  - 赤い破線で基本4線のガイド位置を示す。
- 両手モードでは「右手（1/2）→ 左手（2/2）」と進捗表示。
  1枚目撮影後は同じ画面で2枚目（鏡像）に切り替わる。
- 撮影ボタン押下 → スキャン演出 → 赤ガイドが消え、検出した線が浮かび上がる。

> 本番：ガイドSVGはカメラのライブ映像に半透明で重ねる想定。
> ユーザーは実際の手をガイドに合わせて撮影する。

## [3] 結果

構成（上から）：

1. **ヘッダー**：時間軸ラベル（後天運/先天運）＋「◯本の手相を読み取りました」
2. **写真＋線オーバーレイ**：撮った写真の上に検出線を色分け表示。
   - 検出できた線だけ描画（ない線は引かない）。
   - 線をタップ → その線がハイライト（他は減光）。
3. **線チップ**：検出された線の一覧。タップで選択（写真の線と連動）。
4. **線の解説カード**（選択中の線）：
   - 深掘り解説（特徴の組み合わせ）を冒頭に強調
   - 個別特徴の箇条書き
   - 結婚線なら時期（流年法）
   - ひとことアドバイス（諭し、該当時のみ）
5. **総合**サマリー
6. **左右のちがい**（両手モードのみ）
7. **免責**：娯楽目的の明記 + 採用した見方
8. 操作：「最初から」「もう一度占う」

---

## デザイン方針

- トーン：夜空の藍（深紺）背景 × 手相の線は金色。神秘的で落ち着いた印象。
- AI生成にありがちなクリーム×セリフ体は避ける。
- 線ごとに固有色（生命=金 / 知能=青 / 感情=赤 / 運命=紫 / 太陽=黄 / 財運=緑 / 結婚=桃）。
- モバイル前提の1カラム・縦スクロール。

---

## 検出本数による可変

- 結果は**検出できた線の数だけ**表示する（固定で7枚出さない）。
- extra3が検出されないのは正常。基本4線だけの結果も成立する。
- 1本も特記がない場合のフォールバック文を用意（「穏やかな時期」等）。
