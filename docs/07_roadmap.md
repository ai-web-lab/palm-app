## 最重要リスク：線検出の精度

手相の線をカメラから正確に検出するのは**研究レベルでも難題**。
照明・肌色・しわとの区別が難しく、思った位置に線が引けない可能性が高い。

### 対応オプション
1. **完全自動**：検出をそのまま使う。ズレは「占いだから」と割り切る。
2. **半自動**：ガイド線を表示し、ユーザーが自分の線をなぞる/微調整する。
3. **ハイブリッド（採用）**：MediaPipeで手に追従配置（①）→ ユーザーが
   **結果画面の「線を合わせる」でドラッグ微調整（③）**。`computeLinePoints`が
   線を制御点列で返し、ResultStepでドラッグ編集（`components/ResultStep.tsx`）。
   検出失敗時も中央に初期配置してドラッグで合わせられる。
   ※ 線そのものの実抽出（②, OpenCV.js）は引き続き未実装。

> 推奨：まず基本4線で自動を試し、精度が出なければ半自動にフォールバック。
> 「当てる」より「気づきを与える」アプリなので、多少のズレは許容しやすい。

## extra3線の課題

太陽線・財運線・結婚線は固有の難しさがある（→ palm_rules.json の challenges）。

1. **「ない人」が多い** … 未検出が正常。前向きに解釈する。
   ただし「画質で取れない」と「本当にない」の区別が難しい。
2. **検出が特に難しい** … 短く細く薄い。しわ・影と誤認しやすい。Phase 3送り。
3. **本数カウントが不安定** … 支線やしわを誤算入しやすい。弱い線は除外。
4. **結婚線の流年法は目盛りが流派でバラバラ** … 幅をもった年齢表現に。
5. **extra3の左右差は誤判定しやすい** … 左右差比較は基本4線に限定。
